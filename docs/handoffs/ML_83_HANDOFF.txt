Product and architecture outcome
- Home now uses a Figma-style iPhone-width layout:
  centered 360px max-width content column
  #f7f7f5 background
  NewYork-style "Home" title
  large centered current balance
  equal-width Add and Spend pill buttons
  table-style Today summary
  simplified History header and rows
- The native status bar and existing floating bottom tab bar remain outside
  Home; no Figma chrome was recreated inside the screen.
- Current balance remains:
  sum(active balance entries) - sum(active/non-deleted expense transactions)
- Today summary remains transaction-only and does not include balance additions.
- Home History remains a union feed of expense transactions and balance
  additions, filtered by the selected Home period and sorted newest-first.
- Home History period selector remains limited to:
  Today
  Yesterday
  This week
- Choose date does not appear on Home.

Navigation behavior preserved
- Add routes to /add-balance.
- Spend routes to /add-transaction.
- More routes to Analytics.
- Bottom tabs remain exactly:
  Home
  Analytics & Leaks
  Settings
- Add Transaction, Add Balance, and Shame Card remain root Stack screens, not
  bottom tabs.

History row behavior completed
- Transaction rows:
  show category icon/name
  show time/date
  show one detail line using leak reason first, then note when present
  show signed negative expense amount
  preserve existing PanResponder/Animated swipe behavior
  preserve one-open-row behavior
  preserve edit route and delete confirmation
- Balance rows:
  show a simple non-swipe icon slot
  show balance type name
  use safe fallback "Balance addition" when the type is missing
  show signed positive addition amount
  are not swipeable
  do not expose edit/delete actions
- Home visual badges such as Leak, Normal, and Added were removed from History
  rows to match the Figma list style.

  id,amount,category,isLeak,leakReason,note,createdAt

- Balance additions were not added to Transaction CSV import/export.

  route contract changes were made.
- No new dependencies were added.
- No @expo/ui, @expo/ui-swift-ui, expo-blur, BlurView, Liquid Glass, or
  glass-styles usage was added.
- No service-role/admin Supabase usage was added to mobile app code/config.
