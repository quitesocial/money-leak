ML-62 Local Account Linking v1 Handoff

Summary
- Added native local account linking that runs after successful auth restore or Google login.
- The linking path relinks local SQLite transactions/categories from localOwnerId to the authenticated Money Leak user id from AuthSession.user.id.
- The app remains local-first: no backend tables, RLS, sync, upload, backup, restore, remote merge, account deletion, or login wall was added.

Invariants preserved
- Sign Out clears auth state only; it does not unlink or mutate transaction/category owner_id values.
- Linking updates only rows owned by the current localOwnerId and does not touch rows owned by another owner id.
- Linking preserves row ids, row contents, and soft-deleted transaction deleted_at tombstones.
- Repeated linking is idempotent and does not duplicate rows.
- CSV v1 remains id,amount,category,isLeak,leakReason,note,createdAt.
- Bottom tabs remain Home, Analytics & Leaks, Settings.
- Add Transaction and Shame Card remain root Stack screens.
- No raw owner ids, local owner ids, device ids, access tokens, refresh tokens, provider tokens, OAuth secrets, raw env values, or raw EXPO_PUBLIC_* values are rendered in Account UI.
- Temporary Account diagnostics UI was not restored.
- Google Auth adapter, Supabase config, app.config.js, app.json, and eas.json were not changed.
- No @expo/ui, SwiftUI wrappers, BlurView, expo-blur, glass styling, or Liquid Glass imitation was added.

Implementation notes
- Native linking lives in src/db/account-linking.native.ts and uses SQLite app_metadata, transactions, and categories.
- Web linking returns a skipped result because local SQLite persistence is native-only in this build.
- app_metadata markers account_linked_app_user_id and account_linked_at are non-secret and do not prevent future relinking of new guest rows.
- Auth store sets authenticated state first, then starts linking as a recoverable post-auth operation.
- A linking error keeps the user authenticated and stores safe recoverable copy without raw technical details.
