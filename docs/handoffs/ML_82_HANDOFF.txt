Data model and persistence
--------------------------
New local types:
- src/types/balance.ts

BalanceEntry:
- id: string
- amount: number
- typeId: string
- createdAt: number

BalanceType:
- id: string
- name: string
- createdAt: number
- updatedAt: number
- isDefault: boolean
- isArchived: boolean
- sortOrder: number

Default balance types:
- salary -> Salary
- investment -> Investment
- regalo -> Regalo

SQLite:
- balance_entries table:
  - id TEXT PRIMARY KEY NOT NULL
  - amount REAL NOT NULL
  - type_id TEXT NOT NULL
  - created_at INTEGER NOT NULL
- balance_types table:
  - id TEXT PRIMARY KEY NOT NULL
  - name TEXT NOT NULL
  - created_at INTEGER NOT NULL
  - updated_at INTEGER NOT NULL
  - is_default INTEGER NOT NULL CHECK (is_default IN (0, 1))
  - is_archived INTEGER NOT NULL CHECK (is_archived IN (0, 1))
  - sort_order INTEGER NOT NULL
- balance_types has a unique active-name index on LOWER(name) where
  is_archived = 0.
- Default balance types are seeded through INSERT OR IGNORE in database init.

DB boundaries:
- src/db/balance.ts selects native or web implementation by Platform.OS.
- src/db/balance.native.ts implements native SQLite persistence.
- src/db/balance.web.ts keeps the same native-only stub style as transactions
  and categories.

DB actions:
- getBalanceEntries()
- createBalanceEntry()
- getBalanceTypes()
- createBalanceType()

No edit/delete/archive UI or DB action for balance entries was added in ML-82.
Balance type archive/update was also intentionally not added because it was not
needed for this scope.
