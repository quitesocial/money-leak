ML-81 Add Transaction Wizard - Agent Handoff
Date: 2026-05-28
Project: Money Leak
Stack: Expo / React Native / TypeScript / Expo Router
Version after ML-81: 1.19.0

Status
ML-81 implementation is complete.
Owner manual QA has been completed and passed.
The working tree was clean before this handoff file was added.

What ML-81 Changed
- Add Transaction is now an Add-only route-local wizard at /add-transaction.
- The route remains a pushed root Stack screen, not a bottom tab.
- The native Add Transaction Stack header is hidden so the screen renders the Figma-style in-flow header.
- Edit Transaction remains on the existing shared TransactionForm and was not converted to the wizard.
- Bottom tabs remain Home, Analytics & Leaks, Settings.
- Shame Card remains a pushed root Stack screen.

Wizard Flow
- Details step:
  - Custom header with back button and title "Add Transaction".
  - Amount input autofocus.
  - Decimal numeric keyboard.
  - Amount validation requires a value greater than 0.
  - Supports dot decimals like 12.50 and a single comma decimal like 12,50.
  - Date row uses the existing LocalDatePicker.
  - Type chips: Normal and Leak.
  - Normal proceeds to Category.
  - Leak proceeds to Leak Reason.
- Leak Reason step:
  - Same Add Transaction header.
  - Shows amount, date, and type.
  - Reason chips: Stress, Boredom, Impulse, Habit, Social, Craving.
  - Reason is required before continuing.
  - Switching Leak to Normal clears leakReason and moves to Category.
- Category step:
  - Shows compact summary: amount, date, Normal or Leak / selected reason.
  - Shows only active non-archived categories from the existing local category store.
  - Stores Transaction.category as the stable category ID, not the display name.
  - Category is required before save.
  - Save uses useTransactionsStore.addTransaction.
  - Save payload includes amount, category ID, isLeak, conditional leakReason, note: null, and selected date saved as createdAt epoch milliseconds.
  - After save, behavior is preserved: router.back() when possible, otherwise router.replace('/(tabs)').

Add Category From Add Transaction
- Implemented inside the Add Transaction route as a sub-flow.
- Header title is "Add Category".
- Name input autofocus.
- Save Category uses existing category creation logic/store.
- Validation uses existing validateCategoryName behavior, including empty and duplicate-name errors.
- After successful creation, the flow returns to Category.
- Newly created category is auto-selected when safely found by normalized name.

Explicitly Out Of Scope / Not Changed
- No category icon persistence.
- No icon field added to category model/types.
- No SQLite migrations for category icons.
- No CSV, backup, sync, remote schema, or category data contract changes for icons.
- No fake icon picker UI.
- Add Category in ML-81 is name-only.
- No auth, sync, backup, restore, delete-account, Supabase migration, RLS, remote schema, app icon, splash, or release workflow changes.
- No new dependency was added.
- No @expo/ui, @expo/ui-swift-ui, BlurView, expo-blur, Liquid Glass, or glass styling was added.

Visual Corrections Made After First Implementation

- Bottom CTA/actions were moved into a sticky footer so Next / Save stays at the bottom of the screen.
- The stray small iOS "Next" control was removed by not using returnKeyType="next" on the amount input.
