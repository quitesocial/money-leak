ML-97 [DEV] Home / Analytics tab switch layout jump handoff
Date: 2026-07-09
Project: Money Leak
Version after task: 1.27.6

Summary

ML-97 is implemented, validated, and manually tested. The visible vertical
layout jump during Home <-> Analytics & Leaks tab switching was fixed without
changing the stable Home or Analytics screen layout styles.

The final fix has two parts:

- Keep Home and Analytics warm/attached in the bottom tab navigator:
  detachInactiveScreens={false} on Tabs.
  lazy: false on the Home and Analytics tab screens.
- Remove transient inline refresh status text from Home and Analytics layout
  flow. The focus refresh still runs, but it no longer inserts a short-lived
  "Refreshing..." row that pushes content down for one frame.

Manual QA status

User completed simulator manual testing after the final implementation and
reported everything OK.

The accepted manual behavior includes:

- Home -> Analytics & Leaks no longer has the visible vertical layout jump.
- Analytics & Leaks -> Home no longer has the visible vertical layout jump.
- Fast repeated Home <-> Analytics & Leaks switching stays visually stable.
- The fade/warm screen behavior is acceptable.
- Home layout remains intact.
- Analytics layout, empty state, scroll, and Add Transaction visibility remain
  intact after reverting earlier risky layout experiments.

Important implementation history

Several approaches were tried and rejected before the final fix:

- animation: 'none' alone did not fix the jump.
- A short fade helped visually, but the jump was still visible after the fade.
- Changing Analytics top padding / empty-state spacing was rejected because it
  changed the screen layout and made empty Analytics worse.
- detachInactiveScreens={false} helped but did not fully solve the issue while
  the inline focus-refresh status text still shifted layout.

The final root cause was the focus-refresh state inserting inline refresh text:

- Home rendered home.refreshingTransactions as a Text node in the history group
  when isLoading or isBalanceLoading was true.
- Analytics rendered analytics.refreshing as a Text node before the Overview
  section when isLoading or isBalanceLoading was true.
- During tab focus refresh those rows appeared briefly, pushed content down, and
  disappeared after load completion. This matched the observed "jumps down for a
  fraction of a second, then returns" behavior.

Notes for future agents

- Do not reintroduce inline Home/Analytics focus-refresh status text unless it
  is rendered in a way that cannot affect layout height.
- If future work needs refresh feedback, prefer a non-layout-shifting affordance
  such as an overlay, existing control state, or subtle opacity/state treatment.
- Preserve detachInactiveScreens={false} and lazy: false for Home/Analytics
  unless a future performance task explicitly measures and replaces this
  behavior.
- Do not change Home/Analytics stable layout styles to solve transition issues
  unless the task explicitly scopes a design/layout change.
- If another tab switching jump appears, first check for transient UI inserted
  during focus refresh, not just navigator animation settings.
