ML-94 [DEV] Analytics tab adaptive label handoff
Date: 2026-07-06
Project: Money Leak
Version after task: 1.27.3

Summary

ML-94 is implemented, validated, and manually tested. The Analytics & Leaks
bottom tab label no longer uses hard-coded line breaks. Short localized labels
stay on one line when they fit; longer localized labels can wrap to two lines
inside the existing tab/capsule bounds instead of colliding with neighboring
tabs.

Manual QA status

User completed manual testing after implementation and reported everything OK.
The final manually accepted behavior is adaptive:

- English / short labels stay one line when they fit.
- Longer localized labels, such as Portuguese "Analises e vazamentos", may wrap
  to two lines.
- The label must remain inside the middle tab/capsule area and must not overlap
  Home or Settings.

Key product behavior implemented

- Bottom tabs remain exactly:
  Home
  Analytics & Leaks
  Settings
- The Analytics tab still uses the SF Symbol drop.halffull.
- Active color remains #0088ff.
- Inactive color remains #1a1a1a.
- The floating footer geometry and active rounded capsule animation are
  preserved.
- The Analytics tab label is layout-driven:
  translations do not include forced "\n" line breaks, and React Native decides
  whether the label occupies one or two lines based on available width.
- Add Transaction and Add Balance remain pushed root Stack screens, not bottom
  tabs.

Key implementation files

- app/(tabs)/_layout.tsx
  The custom floating tab bar label now uses numberOfLines={2},
  adjustsFontSizeToFit, minimumFontScale={0.78}, and a constrained maxWidth.
  This keeps short labels on one line while allowing long localized labels to
  wrap safely to two lines.

- src/lib/i18n/translations.ts
  Removed hard-coded newline characters from every tabs.analytics translation.
  The tab label text remains localized but no locale forces a manual line
  break.

- src/lib/__tests__/project-contracts.test.ts
  Updated contract coverage for ML-94 and version 1.27.3. The tests assert that
  the tab label allows up to two lines and that tabs.analytics translations do
  not contain hard newline characters.

- docs/mvp-qa-checklist.md
  Added ML-94 manual QA coverage for short one-line labels, long localized
  two-line fallback, footer navigation, capsule overlap, and Add
  Transaction/Add Balance staying outside bottom tabs.

- package.json / package-lock.json
  Version bumped from 1.27.2 to 1.27.3. app.config.js was not changed and still
  reads version from package.json.
