ML-98 [DEV] Settings - alert for backup/sync handoff
Date: 2026-07-10
Project: Money Leak
Version after task: 1.27.7

Summary

ML-98 replaces the old successful Backup and Sync summary text in Settings
with two independent inline disclosure callouts. These are not native alerts or
modals. The action services, persistence contracts, and sync triggers are
unchanged.

Figma references

- Expanded screen: node 311:2643
- Collapsed screen: node 312:3843
- Expanded Sync callout: node 312:3821
- Expanded Backup callout: node 312:4079
- Collapsed Sync callout: node 312:3857
- Collapsed Backup callout: node 312:4092

Implemented behavior

- Each callout starts collapsed and uses info.circle plus chevron.down.
- Tapping the full white callout toggles its detail and chevron.up/down.
- Each callout is an accessibility button with accessibilityState.expanded.
- Sync and Backup expanded states are independent.
- A new Sync or Backup attempt/result collapses only its corresponding callout,
  including when the new aggregate values match the previous result.
- Sync appears only for the current safe result or a validated persisted safe
  summary from the existing sync metadata boundary.
- Backup appears only after a successful manual backup in the current mounted
  Settings screen. No Backup summary persistence was added.
- Sync detail reports pulled, pushed, applied, conflicts, and ignored totals.
- Backup detail reports transactions, categories, balance types, balance
  entries, and settings from the actual successful result.
- Callout titles, templates, and singular/plural Backup count labels are present
  in every supported runtime locale. Persisted Indian still maps to Hindi UI.
