ML-68 Native Apple Sign-In v1 Handoff

Summary
- Added optional native Apple Sign-In beside Google in Settings -> Account.
- Apple Sign-In is gated by featureFlags.appleAuthEnabled, iOS platform, the
  adapter build/config state, and expo-apple-authentication native availability.
- The implementation uses the native iOS token exchange path:
  expo-apple-authentication -> Apple identityToken ->
  supabase.auth.signInWithIdToken({ provider: 'apple', token, nonce }).
- Guest/local mode remains permanent. There is no login wall.
- Successful Apple login uses the existing authStore setSession flow, so local
  account linking and profile ensure are not duplicated.
- Sign Out behavior remains auth-only and does not mutate local
  transactions/categories.

Implementation notes
- Added expo-apple-authentication and expo-crypto through the Expo-compatible
  install path for SDK 54.
- Added the expo-apple-authentication config plugin to app.json so iOS
  entitlements are generated by Expo prebuild/EAS.
- The Apple adapter generates a cryptographically random raw nonce, hashes it
  with SHA-256 for the native Apple request, and passes the raw nonce to
  Supabase signInWithIdToken.
- Apple cancel returns null and does not save a session.
- Missing identityToken, native/provider failure, and Supabase token exchange
  failure all map to safe generic copy.
- Apple email and fullName are treated as nullable. Private relay email is
  accepted as normal email when Supabase returns it.
- Supabase auth session mapping now infers provider apple/google from
  app_metadata or identities, with google retained as the legacy fallback.
- Settings renders the native AppleAuthenticationButton only in guest mode on
  iOS after adapter availability resolves true.
- Authenticated users see Sign Out only; login buttons are hidden.

Supabase same-email auto-link caveat
- The app does not call linkIdentity, unlinkIdentity, or match Google and Apple
  accounts by email.
- Supabase Auth documentation says OAuth identities with the same verified email
  may be automatically linked on the backend. That behavior is outside this
  client implementation and must be manually verified against the production
  Supabase project before release.

  enabled for the app Bundle ID.
- Confirm Settings guest mode shows Continue with Apple on iOS beside Continue
  with Google.
- Tap Continue with Apple, cancel the native sheet, and confirm Settings returns
  quietly with no error and no saved session.
- Complete Apple login and confirm Settings shows authenticated state and hides
  both login buttons.
- Confirm Apple first-login fullName/email works when Apple provides them.
- Confirm repeat Apple login works when fullName and email are null.
- Confirm Hide My Email/private relay login succeeds.
- Restart after Apple login and confirm authenticated state restores.
- Sign out and confirm local transactions/categories remain visible and are not
  backed up, restored, deleted, unlinked, or synced.

  actual Supabase same-email identity behavior before release.
- Confirm guest mode still works after sign out.
