ML-45 / Epic 45 handoff: iOS Glass Feel v1

Date: 2026-05-13
Repo: Money Leak
App version after work: 1.8.0

Summary
The original ML-45 direction was to add native iOS SwiftUI-backed controls using @expo/ui. That approach was attempted but abandoned because the current @expo/ui SwiftUI beta caused local iOS simulator/dev-build problems and poor layout quality for this app.

The final implemented direction is a conservative React Native-only iOS-like "glass" visual pass. No SwiftUI/native Expo UI wrappers remain in the app. The UI now uses translucent card/control surfaces, softer radii, subtle borders, and iOS-style shadows while keeping the existing React Native controls and product behavior intact.

Important decision
- Do not continue the @expo/ui/SwiftUI beta approach for this pass.
- Do not reintroduce @expo/ui unless a later Expo SDK has stable APIs and the app is explicitly moved to development-build-only UI behavior.
- The final ML-45 implementation intentionally avoids hand-built blur/glass effects and avoids extra blur dependencies. It is an iOS-inspired React Native visual pass, not a native SwiftUI component pass.

Problems seen with @expo/ui
- In Expo Go / normal npm start simulator flow, runtime failed with:
  Cannot find native module 'ExpoUI'
- In npx expo run:ios, the iOS build failed around @expo/ui Swift files:
  SafeAreaControllable is not a member type of ExpoModulesCore.ExpoSwiftUI

  Calling the 'setText' function has failed
