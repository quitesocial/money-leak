Money Leak ML-89 Handoff
Date: 2026-06-11
Status: implemented, manually QA'd, and post-review cleanup completed.

Purpose
-------
This handoff summarizes the completed ML-89 runtime language switch work plus
follow-up fixes, validation, and cleanup so a future AI agent can plan from the
actual current state instead of re-discovering what was already done.

High-Level Result
-----------------
- Settings Language now controls runtime UI language across the app.
- The implementation is dependency-free and local to the existing Expo/React
  Native app.
- English remains the default and safe fallback.
- The existing persisted Settings value "Indian" is preserved and maps to Hindi
  runtime UI copy.
- User-generated values and persisted data contracts were intentionally not
  translated or changed.
- Manual QA was completed by the product owner after implementation and fixes;
  the reported result was OK.

Implemented Scope
-----------------
- Added a dependency-free runtime i18n layer in src/lib/i18n.
- Added useSettingsLanguage with safe English fallback and an in-memory runtime
  notification path for immediate updates after Settings Apply.
- Kept all existing Settings language options selectable.
- Kept the persisted Indian option string and mapped it to Hindi UI locale/copy.
- Wired selected language into:
  - Onboarding.
  - Bottom tab labels and titles.
  - Home.
  - Analytics & Leaks, including filters and ledger display.
  - Settings and option sheets.
  - Add Transaction and Edit Transaction.
  - Add Balance and Edit Balance.
  - Empty, loading, error, success, and confirmation states touched by ML-89.
- Made period/date/month/weekday display language-aware without changing local
  calendar date filtering or period logic.
- Localized system-defined default category and balance type display names.
- Preserved custom category names, custom balance type names, notes, account
  values, profile/account data, IDs, CSV fields, backup/restore DTOs, and sync
  DTOs.
- Updated app/package version from 1.24.1 to 1.25.0.
- Updated docs/mvp-qa-checklist.md with ML-89 manual QA coverage.
- Added a stable AGENTS.md rule for future i18n work:
  route runtime system UI strings through the local i18n layer, and do not
  translate user-generated values or add language fields to persisted models,
  CSV, backup/restore, or sync contracts unless a future task explicitly scopes
  a data-contract migration.

Supported Languages
-------------------
- English: en-GB, default/fallback.
- German: de.
- French: fr.
- Spanish: es.
- Portuguese: pt.
- Italian: it.
- Chinese: zh.
- Russian: ru.
- Indian: hi runtime UI, persisted option remains Indian.

Post-Implementation Fixes
-------------------------
- The initial runtime language wiring translated too little visible UI in some
  screens. This was corrected by expanding runtime coverage and translation
  dictionaries so supported languages are useful across the main app surfaces,
  not limited to a small set of buttons.
- Non-English dictionaries were completed for German, French, Spanish,
  Portuguese, Italian, Chinese, Russian, and Hindi/Indian.
- Translation dictionaries are now typed as complete dictionaries rather than
  loose partials, so TypeScript catches missing keys.
- i18n tests cover completeness and fallback behavior.
- Existing tests that depended on English UI were adjusted to use the i18n
  default path or explicit language mocks where appropriate.

Known Follow-Up Guidance For Future Agents
------------------------------------------
- Use src/lib/i18n/i18n.ts and translation keys for runtime system labels.
- Do not translate user-generated data.
- Do not add language to persisted data contracts unless a future task
  explicitly scopes a data-contract migration.
- If adding new UI strings, add English source copy and complete entries for
  every supported language in src/lib/i18n/translations.ts.
- If adding new default category or balance type labels, keep stable stored IDs
  and translate only display labels.
- Prefer existing shared helpers extracted during cleanup before adding new
  local duplicates:
  - src/lib/amount-input.ts
  - src/lib/date-utils.ts
  - src/lib/swipe-actions.ts
  - src/test-utils/react-test-renderer.ts
