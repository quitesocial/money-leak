ML-81.5 Category Icons Picker Handoff
Date: 2026-05-28
Project: Money Leak
App version after task: 1.20.0

Status
------
ML-81.5 is implemented and manually QA-tested by the user.

The task added persisted local category icons, category icon picking in Add
Transaction, the same create/edit icon flow in Settings > Manage Categories,
and transaction category icons in Home History.

High-level outcomes
-------------------
- Category icons are now a persisted local Category property: iconName.
- Transaction.category still stores only the stable category ID.
- Transaction CSV v1 remains unchanged:
  id,amount,category,isLeak,leakReason,note,createdAt
- Category icons were not added to remote Supabase sync/backup schema.
- No Supabase remote migration was added.
- No auth, backup, restore, sync, delete-account, bottom-tab, or root-stack
  behavior was intentionally changed.
- No new icon dependency was added. Existing expo-symbols / SymbolView is used.
- package.json and package-lock.json were bumped from 1.19.0 to 1.20.0.

User-visible UI implemented
---------------------------
Add Transaction:
- Category step shows each category chip with an inline icon before the name.
- The extra inner circular icon container on category chips was removed to match
  the Figma direction.
- Add Category sub-flow keeps the name input.
- Under "Icon (optional)", the initial state is a single circular plus button.
- Tapping the plus expands an animated icon picker.
- Expanded icon picker uses five icon buttons per row.
- Icon buttons are circular.
- Selected icon state is black circle with white symbol.
- Saving a new category persists the selected iconName.
- If no icon is explicitly selected, fallback icon "tag" is used.
- Newly created category returns to Category step, appears with its icon, and is
  auto-selected.
- Normal and leak transaction creation continue to work.

Settings > Manage Categories:
- The same collapsed-plus / expanded-grid icon picker is now available when
  creating a category from Settings.
- The same picker is available when editing an existing category.
- Category create from Settings saves the selected iconName.
- Category edit from Settings can change iconName.
- Icon-only edits are kept local-only: the DB update changes icon_name but does
  not touch updated_at/source_device_id sync metadata.
- Category name edits still touch sync metadata, preserving the prior remote
  name-sync behavior.

Home History:
- Transaction rows now render the transaction category icon before the category
  name, using the transaction.category ID to resolve the current category icon.
- Unknown or legacy categories fall back safely to the default "tag" icon.

Data model and persistence
--------------------------
Main local Category type:
- src/types/category.ts now includes iconName on CategoryInput/Category.

Central icon registry:
- src/lib/category-icons.ts defines allowed stable icon IDs, labels,
  SF Symbol names, fallback text, default category mappings, and fallback icon.
- Fallback icon is "tag".
- Default category icons:
  - food -> takeoutbag.and.cup.and.straw
  - transport -> bus
  - alcohol -> wineglass
  - shopping -> cart
  - subscriptions -> rectangle.stack
  - other -> bag

SQLite:
- categories table now has icon_name TEXT NOT NULL DEFAULT 'tag'.
- Migration 003_add_category_icons adds icon_name if missing.
- Migration backfills default categories with their default icons.
- Migration keeps custom/legacy categories on safe fallback.
- Legacy category rows without icon_name read safely through fallback/default
  resolution.
- Archived categories are preserved.

Store / DB APIs:
- Category create accepts { name, iconName? }.
- Category update accepts { name, iconName? }.
- updateCategoryName supports optional iconName.
- updateCategoryName supports touchSyncMetadata=false for local-only icon edits.

AGENTS.md update
----------------
Old durable rule:
Keep category data contracts name-only unless a task explicitly introduces
category icons; do not add category icon fields, persistence, schema, sync,
backup, or CSV changes from Figma-only icon visuals.

New durable rule:
Category icons are now a persisted local category property. Keep
Transaction.category storing the stable category ID only. Do not add category
icons to the transaction CSV v1 format or remote sync/backup schema unless a
future task explicitly scopes a data-contract migration.

This belongs in AGENTS.md because it is now a long-lived category data-contract
rule, not an ML-81.5-only detail.

Notes for future planning
-------------------------
- Category icons are local-first. A future task must explicitly scope any remote
  data-contract migration before icons are synced/backed up remotely.
- If future work adds category icon sync, update RemoteCategory,
  Supabase remote tables/migrations, row mappers, backup/restore tests, sync
  conflict behavior, and docs together.
- The shared picker lives in src/components/category-icon-picker.tsx. Prefer
  reusing it instead of creating a second picker implementation.
- The allowed icon list lives in src/lib/category-icons.ts. Prefer adding new
  icons there with stable IDs instead of storing raw glyphs or arbitrary symbol
  names directly from UI code.
- Keep transaction exports/imports on CSV v1 unless a future task explicitly
  scopes a CSV migration.
