What Was Implemented
--------------------
1. Settings layout was rebuilt to match Figma nodes:
   - 203:7919
   - 203:7972
   - 203:8047

2. Settings screen now uses:
   - `#f7f7f5` screen background.
   - Centered max-width 360 content column.
   - NewYork-style title and section typography.
   - Row-based iOS-style sections.
   - Bottom padding that leaves space for the existing floating tab bar.

3. Settings-local reusable UI patterns were added:
   - Title/subtitle rows for reminder, sync, account, delete account.
   - Action/value rows for currency, language, import/export, backup/restore,
     privacy/support.
   - In-screen white bottom option sheet with 38px top corners, grabber,
     circular xmark/checkmark buttons, staged selection, cancel/apply behavior.

4. Local Settings preferences were added under `src/lib/settings-preferences.ts`:
   - Language preference, default `English`.
   - Currency preference, default `Euro`.
   - Foreground auto-sync preference, default enabled only when incremental sync
     is feature-enabled.

5. Foreground auto-sync was wired to the new local preference:
   - Only affects `useForegroundAutoSync`.
   - Still requires authenticated user context.
   - Still respects existing throttle metadata.
   - Still routes through the existing sync service boundary.
   - Still uses `source: 'foreground'`.

6. Category management was moved into Settings as the primary category UI:
   - Uses `useCategoriesStore`.
   - Uses `useCategoriesRefresh`.
   - Uses `CategoryIconPicker`.
   - Uses existing `validateCategoryName`.
   - Uses existing `getArchiveCategoryError`.
   - Add/edit inline state preserves existing validation and icon picker
     behavior.
   - Delete remains archive-only with existing confirmation and existing
     `Other`/last-category protections.
   - The existing `/categories` route may remain, but Settings no longer
     depends on it for primary category management.

7. Category swipe behavior was corrected after manual Figma review:
   - Figma reference: node `203:7937` inside `203:7919`.
   - At rest, category rows are not white; they sit on the Settings background.
   - On swipe, the row content becomes white.
   - Swipe right reveals the red delete action on the left.
   - Swipe left reveals the green edit action on the right.
   - The content does not translate left/right like transaction history cards.
     Instead, it shrinks from 360px to about 300px, revealing a 52px action
     plus 8px gap.
   - On right/delete reveal, only the left content corners are rounded.
   - On left/edit reveal, only the right content corners are rounded.
   - This behavior intentionally differs from transaction row translation,
     because the Settings Figma category state shows shrink/reveal behavior.

8. General rows preserve existing behavior:
   - Sign out calls only the auth store `signOut`.
   - Delete account keeps existing confirmation/service behavior.
   - Import/export/backup/restore/manual sync keep existing service calls,
     loading states, success summaries, and generic safe failures.
   - Privacy/support links use centralized `APP_LINKS` and generic link
     failure copy.

9. Route and chrome ownership were preserved:
   - Settings does not recreate status bar.
   - Settings does not recreate the tab bar.
   - Bottom tabs remain exactly Home, Analytics & Leaks, Settings.
   - Existing pushed root routes remain root Stack routes.

10. Version metadata was updated for release:
   - `package.json` version is now `1.24.0`.
   - Root `package-lock.json` version entries are now `1.24.0`.
   - `app.config.js` still resolves Expo `version` from `package.json`.
   - No dependency versions were changed.

Notes For Future Planning
-------------------------
1. Currency and language are still local Settings preferences only. Do not plan
   FX conversion or app-wide i18n as part of ML-87 unless a new task explicitly
   scopes it.

2. Settings category UI is now the primary category management surface. The old
   `/categories` route can remain unless a future task explicitly removes or
   repurposes it.

3. Foreground auto-sync must remain gated by:
   - `featureFlags.incrementalSyncEnabled`
   - authenticated user context
   - existing throttle metadata
   - existing sync service boundary
   - `source: 'foreground'`

4. Do not move sync calls directly into screen code or bypass manual sync /
   metadata service boundaries.

5. Do not change CSV, backup/restore/sync schemas, app config, or dependencies
   when building on top of this work. The ML-87 version bump has already been
   applied as `1.24.0`; do not bump it again unless a future release task
   explicitly scopes another version change.
