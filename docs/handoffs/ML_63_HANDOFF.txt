ML-63 Supabase Backend User Profile / RLS Foundation v1 Handoff

Summary
- Added the first Supabase backend identity foundation for Money Leak.
- Added a remote profiles table migration with RLS policies that restrict each
  authenticated user to their own profile row.
- Added an app-side Supabase profile service that upserts/selects one profile
  for the token-free authenticated Money Leak session.
- Wired profile ensure after successful auth restore and Google login as a
  recoverable post-auth task.
- The app remains local-first: profile failure does not sign out the user,
  block navigation/startup, hide local data, or start backup/sync.

Migration
- supabase/migrations/20260520000000_create_profiles.sql
- Creates public.profiles with id, email, display_name, avatar_url, provider,
  created_at, and updated_at.
- Enables Row Level Security.
- Grants authenticated select, insert, and update only through own-row policies.
- Revokes anonymous/public table access.
- Adds an updated_at trigger.
- Does not create remote transaction, category, settings, backup, restore, or
  sync tables.

App changes
- Added src/lib/supabase/supabase-profile-service.ts.
- Added ensureUserProfile(sessionOrUser), Profile, and ProfileResult types.
- Uses the existing Supabase anon client only; no service role key is used.
- Stores only non-sensitive profile fields from the existing auth session:
  email, display name, avatar URL, and provider.
- Returns safe recoverable failure results without raw backend error details.
- Updated src/store/auth-store.ts to inject and run profile ensure after
  authenticated state is established.
- Profile ensure is independent from the existing local account linking path.

Tests
- Added src/lib/supabase/__tests__/supabase-profile-service.test.ts.
- Covered authenticated upsert/select, unauthenticated skip, unavailable client,
  Supabase errors/throws, repeated ensure idempotent app behavior, sensitive
  session fields not being persisted/returned, and no local data store actions.
- Updated src/store/__tests__/auth-store.test.ts for profile ensure after
  restore/login and recoverable profile failure behavior.
