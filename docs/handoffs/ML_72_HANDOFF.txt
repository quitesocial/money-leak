ML-72 Manual Sync Now UI v1 Handoff

Summary
- Added a user-facing manual Sync card in Settings for authenticated users when
  featureFlags.incrementalSyncEnabled is enabled.
- Enabled featureFlags.incrementalSyncEnabled for this release.
- Sync remains manual-only. No automatic sync was added on app start, login,
  session restore, sign out, background behavior, or transaction/category
  mutation.

Implemented behavior
- Settings shows the Sync card only for authenticated users while the sync flag
  is enabled.
- Guest/local mode does not show the Sync card and cannot trigger sync.
- The Sync button label is "Sync now" when idle and "Syncing..." while the
  manual sync service is running.
- The Sync button is disabled while syncing.
- Pressing Sync now calls src/lib/sync/manual-sync-service.ts through
  manualSyncService.runIncrementalSync with the current auth context.
- Success renders only safe aggregate counts:
  pulled, pushed, applied, conflicts, and ignored.
- Failure, skipped results, and thrown errors render only:
  "Couldn't sync. Try again."
- Last sync time is read through the existing sync metadata boundary and is
  shown as "Last sync:" when a valid timestamp exists.
- Existing backup, restore, delete account, import/export, auth, and navigation
  behavior was left unchanged.

Out of scope / intentionally not added
- No auto-sync on app start, login, session restore, sign out, background, or
  local transaction/category changes.
- No conflict UI.
- No category tombstone semantics beyond the existing safe ignore/count
  behavior.
- No backup/restore behavior change.
- No Delete Account behavior change.
- No CSV v1 change.
- No navigation or bottom tab change.
- No raw diagnostics UI.
- No service-role/admin Supabase code in the mobile app.
- No new dependencies.
- No @expo/ui, @expo/ui-swift-ui, BlurView, expo-blur, Liquid Glass, or glass
  styling.

Pre-Codex baseline already completed by owner

- Sign Out does not delete local data.

Manual QA checklist
- In guest/local mode, open Settings and confirm the Sync card is hidden.
- Sign in with Google, open Settings, and confirm the Sync card is visible.
- Tap Sync now and confirm the button changes to Syncing..., then returns to
  Sync now.
- Confirm successful sync shows only aggregate counts and, when available,
  Last sync.
- Force a recoverable sync failure or missing session and confirm Settings
  shows only "Couldn't sync. Try again."
- Tap Sign Out and confirm sync does not run and local transactions/categories
  remain visible after returning to guest/local mode.
- Run manual backup, restore, and Delete Account flows and confirm they do not
  trigger sync.

Preserved invariants
- Incremental sync remains behind featureFlags.incrementalSyncEnabled.
- Incremental sync is still not started automatically by auth/session/app
  lifecycle or local data mutations.
- Guest/local mode remains available with no login wall.
- Mobile app code/config does not use Supabase service-role/admin clients.
- Sync UI uses the manual sync service and existing sync metadata boundary, not
  remote adapters or Supabase clients directly.
