ML-96 [DEV] Home transaction feed date headers handoff
Date: 2026-07-08
Project: Money Leak
Version after task: 1.27.5

Summary

ML-96 is implemented, validated, and manually tested. Home now shows localized
date headers in the transaction feed:

- Today shows one full-date header above rows when the selected period has
  visible rows.
- Yesterday shows one full-date header above rows when the selected period has
  visible rows.
- This week groups the Home union feed by local calendar day, newest day first.
- Expense transactions and balance additions from the same local day share one
  date header.
- Empty selected periods keep the existing empty state and do not render an
  orphan date header.

Manual QA status

User completed manual testing after implementation and reported everything OK.
The manually accepted behavior includes:

- Today date header renders correctly on Home.
- Yesterday date header renders correctly on Home.
- This week shows multiple local-day groups in newest-first order.
- Mixed expense transaction and balance addition rows from the same day appear
  under one shared date header.
- Home transaction swipe edit/delete still works.
- Home balance addition swipe edit/delete still works.
- Empty period state does not show an orphan date header.
- Home still does not show Choose date.
- Navigation, CSV, sync/backup/restore, and data-contract behavior were not
  regressed.

Key product behavior implemented

- Home feed remains the same union feed of expense transactions and balance
  additions.
- Home period selector remains limited to:
  Today
  Yesterday
  This week
- Home feed filtering still uses the existing local calendar period helpers.
- Home feed sorting remains newest-first before grouping.
- Date grouping is a render-layer concern and does not change stored
  timestamps, Transaction data, Balance data, CSV contracts, sync DTOs, backup
  DTOs, or restore DTOs.
- Date labels use the existing i18n date formatter and language locale support.
  English labels use the Figma-style long date format, for example:
  23 April 2026.
- Date headers render between the Transactions / More header row and the first
  row/group.
- Existing Home row components and behaviors are preserved:
  expense rows use SwipeableTransactionRow;
  balance rows keep Home-local shrink-to-reveal swipe behavior.

Notes for future agents

- Treat src/features/home/home-history-sections.ts as the Home-specific
  grouping boundary. Feed it already-filtered and already-sorted Home feed
  items.
- Keep date grouping local-calendar based. Do not switch Home grouping to UTC.
- Keep Home period options limited to Today, Yesterday, and This week unless a
  future task explicitly expands Home period scope.
- Keep date headers out of empty states.
- Keep LedgerTransactionRow presentation-only. Do not put CRUD, routing, store,
  or Alert logic inside it.
- Keep expense transaction rows on SwipeableTransactionRow and the transaction
  domain path:
  /transaction/[id]/edit and removeTransaction.
- Keep balance addition rows on the balance domain path:
  /balance/[id]/edit and removeBalanceEntry.
- Preserve the shrink-to-reveal swipe pattern for Home rows. Do not return to
  translating the foreground row out of view.
- If future Home feed behavior changes, retest Today, Yesterday, This week,
  mixed expense/balance grouping, empty states, and transaction/balance swipe
  regressions together.
