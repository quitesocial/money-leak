Money Leak ML-88 Handoff
Date: 2026-06-10

Purpose
-------
This handoff summarizes the completed ML-88 currency preference work plus the
dead route/code cleanup that happened during the same thread. It is intended for
the next AI agent so they know what is already done and what product/data
contracts must remain intact.

Completed ML-88 Scope
---------------------
- Settings currency is now a display-only app preference.
- No FX conversion was added.
- No amount mutation was added.
- No per-transaction currency or per-balance currency was added.
- No Transaction CSV v1 schema change was added.
- No sync, backup, restore, remote DTO, or Supabase schema migration was added.
- The ML-87 Settings layout and staged currency sheet behavior were preserved.
- No separate ML-88 Figma link/node was found in the supplied task.

Currency Formatting Behavior
----------------------------
- Currency display is centralized in src/lib/display-formatters.ts.
- Currency metadata lives in src/lib/settings-preferences.ts.
- Added getCurrencySymbol(currency).
- Added formatMoneyAmount({ amount, currency, sign, useGrouping,
  spaceBeforeCurrency }).
- formatEuro remains as a backward-compatible Euro wrapper.
- Euro remains the default.
- Every supported currency renders with two decimals, including JPY.
- Invalid or non-finite numeric values format safely as 0.00.
- Signed labels use the provided + or - sign without double-signing.
- Analytics-style grouped money labels keep spaced thousands when useGrouping is
  true.
- Currency symbols are suffix-style after the amount, preserving the existing
  Euro style.
- Symbol map used:
  Euro -> EUR suffix symbol
  US Dollar -> $
  Canadian Dollar -> C$
  Australian Dollar -> A$
  Russian Ruble -> RUB suffix symbol
  Indian Rupee -> INR suffix symbol
  Chinese Yuan -> CN¥
  British Pound -> GBP suffix symbol
  Japanese Yen -> JPY suffix symbol

Runtime Currency Wiring
-----------------------
- Added a small useSettingsCurrency hook in src/lib.
- The hook reads getSettingsCurrency() on mount and screen focus.
- The hook safely falls back to Euro on read failure.

  - Home Today summary
  - Home History balance addition labels
  - Home History expense labels
  - Analytics ledger Added/Spent labels
  - Analytics filtered rows and all period modes
  - Add Transaction amount suffix

- Edit Transaction currently has no money preview/suffix in the UI, so it was
  intentionally left visually unchanged.

Important Guidance For Future Planning

  migration.

  getCurrencySymbol from src/lib/display-formatters.ts rather than ad hoc
  formatting.
