ML-99 [DEV] Settings - anonymous feedback form handoff
Date: 2026-07-13
Project: Money Leak
Version after task: 1.28.0
Status: Implemented and automated validation passed; owner actions and native manual QA remain required

Summary

ML-99 adds a guest- and authenticated-accessible Leave Feedback CTA to the
bottom of Settings, an opaque React Native feedback bottom sheet based on the
provided Figma states, and a real anonymous insert-only Supabase submission
path. Feedback is not stored locally and is not connected to auth, sync,
backup, restore, CSV, transaction, balance, or category contracts.

Implemented behavior

- Settings renders one full-width black Leave Feedback pill after the existing
  sections and before the existing floating-tab bottom spacing.
- The form uses Modal, Animated, KeyboardAvoidingView, Pressable, TextInput,
  expo-symbols star/star.fill, safe-area padding, an opaque #f7f7f5 surface,
  rounded top corners, and shadow. No blur/glass or new UI dependency was added.
- The form starts with five outline stars and no label. Ratings 1-5 fill the
  corresponding stars and show localized Hated it / Didn't like it / It was
  okay / Liked it / Loved it copy.
- Rating is required. Comment is optional, multiline, capped at 2000 input
  characters, trimmed by the service, and stored as null when blank.
- Stars expose button role, localized labels, and selected accessibility state.
  The comment, Submit button, CTA, and backdrop close action are labeled.
- A synchronous in-flight guard and disabled controls prevent fast duplicate
  submits. Backdrop close and Android Back are ignored during submission.
- Failure keeps the form open and renders only generic localized copy. Success
  closes and resets the form and shows a localized thank-you Alert. Reopening
  always starts clean.
- All new UI and accessibility strings exist in English, German, French,
  Spanish, Portuguese, Italian, Chinese, Russian, and Hindi runtime UI for the
  persisted Indian option.

Service and privacy boundary

- FeedbackSubmissionPayload contains exactly rating, nullable comment,
  app_version, platform, and language.
- App version comes from Constants.expoConfig.version, platform from
  Platform.OS, and language from the current runtime Settings language.
- The service validates rating, comment length, and bounded non-empty metadata
  before calling the adapter.
- The adapter uses the existing public Supabase client and performs only
  from('feedback_submissions').insert(payload). It never selects a created row.
- Raw errors, comments, payloads, response data, URLs, env values, tokens, and
  identifiers are not logged or returned to the UI.
- No email, auth user ID, ownerId, localOwnerId, deviceId, token, transaction,
  balance, category, amount, or other financial data is attached to feedback.

Supabase migration and required manual action

Migration file:
supabase/migrations/20260713000000_create_feedback_submissions.sql

The migration creates public.feedback_submissions with server-generated UUID
and created_at values, rating 1-5, nullable comment <= 2000, and bounded app
version/platform/language fields. It enables RLS, revokes all privileges from
public/anon/authenticated, then grants only INSERT to anon and authenticated.
One INSERT policy repeats the safe field checks. No SELECT, UPDATE, or DELETE
grant or policy is created.
